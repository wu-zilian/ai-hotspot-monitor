## 1. 项目概述

### 1.1 项目简介

AI热点监控工具是一个智能化的热点发现和通知系统，通过整合爬虫技术、AI内容分析和多渠道通知，帮助用户实现7×24小时自动化热点监控。

### 1.2 核心价值

- **自动化发现**：减少人工搜索时间，提高效率
- **AI验证**：智能识别虚假内容，确保信息真实性
- **实时通知**：第一时间获取热点信息
- **多源监控**：覆盖多个科技新闻和信息源

### 1.3 目标用户

- AI编程博主
- 科技媒体从业者
- 投资人和研究人员
- 企业技术团队

### 2.2 技术栈

#### 后端技术栈
- **运行时环境**: Node.js 18+
- **Web框架**: Express.js 4.18
- **数据库**: MongoDB 7.0
- **认证**: JWT (jsonwebtoken)
- **密码加密**: bcryptjs
- **数据验证**: Joi
- **定时任务**: node-cron
- **爬虫**: axios + cheerio
- **实时通信**: WebSocket (ws)
- **邮件服务**: nodemailer
- **日志记录**: winston
- **安全**: helmet, express-rate-limit

#### 前端技术栈
- **框架**: React 18.2
- **语言**: TypeScript 5.3
- **UI库**: Ant Design 5.12
- **状态管理**: React Context
- **路由**: React Router 6.20
- **HTTP客户端**: axios 1.6
- **图表**: Recharts 2.10
- **构建工具**: Create React App

#### AI服务集成
- **OpenRouter API**: 支持GPT-4、Claude等多种AI模型
- **DeepSeek API**: 国内AI服务，访问稳定、价格友好
- **灵活切换**: 通过环境变量一键切换AI服务商

## 4. 数据库设计

### 4.1 数据库集合

| 集合名 | 用途 | 主要索引 |
|--------|------|----------|
| users | 用户信息 | email (unique), username (unique) |
| keywords | 关键词管理 | userId, name, isActive |
| hotspots | 热点内容 | userId, timestamp, isVerified, keywords |
| tasks | 定时任务 | userId, isActive |
| notifications | 通知记录 | userId, isRead, createdAt |

### 4.2 索引设计

```javascript
// users集合索引
db.users.createIndex({ email: 1 }, { unique: true })
db.users.createIndex({ username: 1 }, { unique: true })

// keywords集合索引
db.keywords.createIndex({ userId: 1, isActive: 1 })
db.keywords.createIndex({ name: 1 })

// hotspots集合索引
db.hotspots.createIndex({ userId: 1, timestamp: -1 })
db.hotspots.createIndex({ keywords: 1 })
db.hotspots.createIndex({ isVerified: 1 })

// notifications集合索引
db.notifications.createIndex({ userId: 1, isRead: 1, createdAt: -1 })

// tasks集合索引
db.tasks.createIndex({ userId: 1, isActive: 1 })
```

## 8. 部署方案

### 8.1 Docker部署

```yaml
# docker-compose.yml
services:
  mongodb:
    image: mongo:latest
    ports:
      - "27017:27017"
    environment:
      MONGO_INITDB_ROOT_USERNAME: admin
      MONGO_INITDB_ROOT_PASSWORD: admin123
    volumes:
      - mongodb_data:/data/db

  backend:
    build: ./backend
    ports:
      - "5001:5001"
    depends_on:
      - mongodb
    environment:
      - NODE_ENV=production

  frontend:
    build: ./frontend
    ports:
      - "3000:80"
    depends_on:
      - backend
```

### 8.2 环境变量配置

```env
# AI服务配置
AI_SERVICE_PROVIDER=deepseek  # 或 openrouter
DEEPSEEK_API_KEY=sk-your-deepseek-key
```
