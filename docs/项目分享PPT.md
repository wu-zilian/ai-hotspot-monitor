## 1. 项目背景与痛点

### 现状问题

- **效率低下**：人工搜索热点需要浏览多个网站，耗时耗力
- **容易遗漏**：无法实现7×24小时持续监控，错过重要信息
- **虚假内容**：AI编程领域虚假信息多，难以辨别真伪
- **通知延迟**：无法第一时间获取热点，错失最佳时机

### 目标用户

👤 **AI编程博主** - 需要持续关注行业动态
👤 **科技媒体从业者** - 第一时间获取新闻热点
👤 **投资人/研究人员** - 了解技术发展趋势
👤 **企业技术团队** - 跟踪前沿技术

---

## 2. 解决方案概述

### 系统定位

AI热点监控工具是一个**智能化的热点发现和通知系统**，通过整合爬虫技术、AI内容分析和多渠道通知，帮助用户实现**7×24小时自动化热点监控**。

### 核心价值

- **自动化发现** - 减少人工搜索时间，提高效率
- **AI验证** - 智能识别虚假内容，确保信息真实性
- **实时通知** - 第一时间获取热点信息
- **多源监控** - 覆盖多个科技新闻和信息源

## 4. 技术架构设计（续）

### 4.2 数据模型设计

| 模型 | 用途 | 主要字段 |
|------|------|----------|
| **User** | 用户信息 | username, email, password, role |
| **Keyword** | 关键词管理 | name, description, isActive |
| **Hotspot** | 热点内容 | title, content, source, confidence |
| **Task** | 定时任务 | name, schedule, keywords, isActive |
| **Notification** | 通知记录 | title, content, type, isRead |

### 4.3 核心流程

```
用户添加关键词 → 定时任务触发 → 爬虫采集数据
        ↓
AI内容分析 → 可信度评分 → 热点存储 → 多渠道通知
        ↓
用户查看热点 → 验证真实性 → 跟踪关注
```

### 5.3 AI服务集成

#### OpenRouter API
- **模型**：GPT-4、Claude等
- **优势**：模型选择丰富
- **适用**：国际用户

#### DeepSeek API
- **模型**：deepseek-chat
- **优势**：访问稳定、价格友好
- **适用**：国内用户

#### 切换方式
```env
AI_SERVICE_PROVIDER=deepseek  # 或 openrouter
DEEPSEEK_API_KEY=sk-xxx
OPENROUTER_API_KEY=sk-or-v1-xxx
```

## Q&A

### 常见问题

**Q: 如何获取AI API密钥？**
A: OpenRouter访问 openrouter.ai，DeepSeek访问 deepseek.com

**Q: 系统支持哪些邮箱服务？**
A: 支持QQ邮箱、Gmail、网易邮箱、Outlook等主流邮箱

**Q: 如何提高监控效率？**
A: 合理设置扫描间隔、优化关键词、提高置信度阈值

**Q: 是否支持多用户？**
A: 支持，每个用户独立配置和管理自己的关键词

**Q: 数据存储在哪里？**
A: 支持本地MongoDB或云数据库部署
