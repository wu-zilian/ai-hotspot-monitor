# 扫描任务状态跟踪功能

## 🎯 新增功能

### 功能描述
点击"后台运行"后，可以在Dashboard上随时查看扫描任务进度，不会再丢失任务状态。

## 📋 使用流程

### 1. 启动扫描
```
点击"立即扫描" → 确认扫描 → 开始扫描
```

### 2. 查看进度
```
进度Modal自动打开 → 实时显示5个阶段进度
```

### 3. 后台运行
```
点击Modal中的"后台运行"按钮
```

### 4. 重新打开进度 ⭐ 新功能
后台运行后，有两个方式重新打开进度：

#### 方式1: 点击状态卡片
Dashboard顶部会出现蓝色提示卡片：
```
┌─────────────────────────────────────────┐
│ 🔄 扫描任务进行中                       │
│ 状态: AI分析中...  进度: 45%            │
│                            [查看进度]    │
└─────────────────────────────────────────┘
```

点击"查看进度"按钮即可重新打开进度Modal

#### 方式2: 点击列表标题
在"最新热点"卡片标题处显示：
```
最新热点 后台扫描中 (点击查看)  ← 点击这里
```

点击蓝色标签即可重新打开进度Modal

## 📊 任务状态说明

| 状态 | 显示文本 | 说明 |
|------|----------|------|
| pending | 等待中 | 任务已创建，等待执行 |
| crawling | 正在爬取数据... | 从信息源获取热点 |
| crawled | 爬取完成 | 数据获取完毕 |
| analyzing | AI分析中... | 正在进行内容验证 |
| completed | 已完成 | 任务成功完成 |
| failed | 失败 | 任务执行失败 |

## 🔄 轮询机制

- 系统每3秒自动检查一次任务状态
- 更新状态卡片上的进度信息
- 任务完成后自动停止轮询

## 💡 使用场景

### 场景1: 后台运行其他任务
1. 启动扫描
2. 点击"后台运行"
3. 继续浏览热点列表
4. 时不时点击"查看进度"按钮检查扫描状态

### 场景2: 长时间扫描
1. 启动大规模扫描（可能需要几分钟）
2. 关闭进度Modal
3. 做其他工作
4. 回来后点击"查看进度"
5. 看到实时进度和已完成的分析结果

### 场景3: 刷新页面后
1. 扫描进行中刷新了页面
2. 看到任务状态卡片还在
3. 点击"查看进度"继续监控

## 🎯 功能优势

| 优化前 | 优化后 |
|--------|--------|
| ❌ 点击"后台运行"后无法查看进度 | ✅ 随时点击状态卡片查看进度 |
| ❌ 不知道任务是否还在运行 | ✅ 实时显示任务状态 |
| ❌ 刷新页面后丢失任务 | ✅ 状态卡片持久显示 |
| ❌ 不知道进度到哪里了 | ✅ 显示百分比和当前阶段 |

## 📝 注意事项

1. **状态卡片自动隐藏**
   - 任务完成后自动隐藏
   - 任务失败后显示错误信息

2. **手动关闭**
   - 点击卡片右上角的 × 可关闭
   - 关闭后仍可通过列表标题重新打开

3. **实时更新**
   - 进度信息每3秒自动更新
   - 重新打开Modal时显示最新状态

4. **WebSocket优先**
   - 如果Modal保持打开，使用WebSocket实时推送
   - 关闭Modal后使用轮询检查状态
